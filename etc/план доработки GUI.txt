План доработки GUI сервиса
--------------------------

1) В редакторе IDE, при нажатии кнопки «Настроить конфигурацию запуска»,
в диалоговом окне настройки после поля «Опции» добавить два поля:

-- Применить темплет: [выпадающий список]
-- Для файла: [текстовый редактор]

2) На панель инструментов редактора IDE добавить: 
-- выпадающий список выбора вариантов кода Design или Deploy
-- кнопку «Синхронизировать темплет» (можно нарисовать на ней стрелки направо/налево)
-- при синхронизации мои консольные утилиты порождают некоторый выход, направить его в окно, расположенное под окнами редакторов кода

3) При создании проекта из шаблона 
-- добавить поля «Применить темплет» и «Для файла» в диалог «Новый шаблон», аналогично п.1 
-- копировать атрибуты «Применить темплет» и «Для файла» в настройки проекта

Как это должно работать:

При нажатии «Синхронизировать темплет» происходит сохранение и преобразование файла, указанного в поле "Для файла"
согласно шаблону. Я написал шаблоны Design|Deploy для "портфеля задач". Можно посмотреть, как они выглядят здесь:
github.com/templet-language/newtemplet/src/tridentity.

При нажатии "Запустить" предварительно выполняется преобразование файла, указанного в поле "Для файла",
согласно темплету в вариант представления кода Deploy.

Всё это выполняется консольными утилитами на стороне сервиса. Параметры утилит: обрабатываемый файл, темплет и параметр Design/Deploy.

Для удобства разработки также добавить кнопку "Компилировать". При её нажатии выполняется загрузка на управляющий узел кластера 
текущей конфигурации кода (Design или Deploy) сборка, формирование выходного log-файла и его выгрузка с управляющего узла кластера.
Содержимое log-файла направить его в окно, расположенное под окнами редакторов кода.
